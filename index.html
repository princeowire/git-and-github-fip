💻 Git & GitHub Guide

🧠 What is Git?

Git is a distributed version control system that tracks changes in source code during software development.

It allows multiple developers to work on the same codebase without overwriting each other's changes.

git init
git add .
git commit -m "Initial commit"

🌐 What is GitHub?

GitHub is a web-based platform built on top of Git. It allows teams to collaborate on code by hosting repositories in the cloud.

It adds features like pull requests, issues, project boards, and CI/CD integrations.

🌿 Git Branches

Branches allow you to diverge from the main codebase and work independently.

git branch feature-x
git checkout feature-x
# or simply:
git checkout -b feature-x

Once your feature is ready, you can merge it into the main branch:

git checkout main
git merge feature-x

🧬 Cloning a Repository

Cloning means copying a GitHub repository to your local machine.

git clone https://github.com/username/repo.git

This creates a folder with all the repo's code and history.

🔁 Push vs Pull

📤 git push

Push sends your local commits to the remote repository (usually GitHub).

git push origin main

This command uploads your latest commits to the main branch on GitHub.

📥 git pull

Pull fetches new commits from the remote repository and merges them into your local branch.

git pull origin main

It’s a combination of git fetch and git merge.

💡 Tip: Always pull before pushing to avoid conflicts!

📤 Pull Requests

A Pull Request (PR) lets you propose changes to a repository. You can review, comment, and approve changes before merging.

Steps:

  1. Fork the repo
  2. Create a branch
  3. Push your changes
  4. Create a PR on GitHub

🌱 Merging Branches

Merging combines changes from one branch into another, usually a feature branch into main.

git checkout main
git merge feature-branch

If there are no conflicts, the branches are merged automatically.

💡 Best Practice: Always test your feature branch before merging!

⏪ Reverting a Pull Request

Sometimes a pull request (PR) is merged but later causes bugs. You can revert it.

🔧 Method 1: Use GitHub Revert Button

Go to the merged pull request page and click Revert. GitHub creates a new PR to undo the changes.

🔧 Method 2: Manual Revert in Git

Find the PR merge commit hash, then run:

git revert COMMIT_HASH

This undoes the PR changes by creating a new reverse commit.

📦 Git Stash

Stash temporarily shelves changes you’re not ready to commit.

git stash
git pull origin main
git stash apply

🧯 Common Git & GitHub Problems + Fixes

1. ❌ Accidentally Committed the Wrong File

Problem: You committed a large file or secret key.

Fix:

git reset HEAD~1
# unstages the last commit but keeps changes

# or delete file and recommit
rm secret.txt
git add .
git commit -m "remove secret"

2. 🔄 Merge Conflicts

Problem: Git can't auto-merge files when two branches change the same lines.

Fix: Open the conflicted file. Git will show markers like:

<<<<<<< HEAD
Your changes
=======
Their changes
>>>>>>> branch-name

Manually fix, then:

git add .
git commit

3. 🔁 Pushed to the Wrong Branch

Problem: You accidentally pushed to main instead of a feature branch.

Fix:

git checkout -b correct-branch
git push origin correct-branch

# (optional) revert main if needed:
git checkout main
git revert HEAD

4. 📦 Forgot to Pull Before Push

Problem: Push is rejected because your local branch is behind.

Fix:

git pull --rebase origin main
# then push again:
git push origin main

5. 🧨 Detached HEAD

Problem: You checked out a specific commit and now you're in a detached state.

Fix:

git checkout -b new-branch

6. 🔐 GitHub Authentication Issues

Problem: GitHub rejects your push due to authentication or token issues.

Fix:

  • Use a Personal Access Token (PAT) instead of a password.
  • Configure GitHub CLI or SSH keys.
git remote set-url origin https://TOKEN@github.com/username/repo.git
